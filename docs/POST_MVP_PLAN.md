## 1. Nice-to-have (после v1)

Кандидаты в порядке ориентировочной ценности/дешевизны. Не роадмап с датами —
пул, из которого берётся следующая задача по желанию/спросу.

| Фича | Что даёт | Обучающая нагрузка Go |
|---|---|---|
| **Стриминг ответа LLM** (токен-за-токеном в терминал) | Отзывчивость на длинных ответах | Разбор SSE-стрима: `data:`-чанки до `[DONE]`, `bufio.Scanner`, потоковый рендер (`internal/llm/stream.go`) |
| **Горутинный worker-pool сбора диффов** | Ускорение `digest`/`review` на многих коммитах/файлах | Каналы, `sync.WaitGroup`/`errgroup`, `-race`-чистота, backpressure |
| **Кастомные шаблоны промптов/вывода** | Команды подгоняют формат под себя | `text/template`, `//go:embed` дефолтов, override из конфига |
| **`gitl digest --tui`** (bubbletea) | Интерактивная навигация по коммитам/файлам, свёртка секций | The Elm Architecture поверх готовых артефактов (`internal/tui`) |
| **Кэш ответов LLM** по хэшу диапазона (`~/.cache/gitl/`) | Не переплачивать за повтор одного диапазона | Хэширование входа, `os.UserCacheDir()`, инвалидация, атомарная запись |
| **Полный SLSA provenance** | Верифицируемая цепочка сборки (не только подпись) | GitHub Actions provenance, SLSA-генераторы; следующий шаг после базового cosign |

## 2. Опциональная монетизация (по желанию, не цель)

Основная «валюта» проекта — освоенные Go-навыки, портфолио и рабочий инструмент,
которым автор пользуется сам. Доход — осознанно **бонус**, а не KPI.

- **Платный GitHub Marketplace Action** — самый естественный платный тир, раз
  Action уже часть MVP: например, бесплатно — `review`/`changelog` с базовым
  риск-скорингом; платно — мульти-репо digest, кастомные политики, приоритетные
  модели. **Биллинг целиком на стороне GitHub Marketplace** — не требует своей
  платёжной инфраструктуры и общения с покупателями.
- **GitHub Sponsors / «Buy me a coffee»** — кнопка в README. Пассивно, ноль общения.
- **Спонсорские фичи** — «профинансируй фичу» через issues с меткой. Пассивно.

Реалистичная оценка: для нишевого dev-инструмента доход, скорее всего, скромный.

## 3. Кандидаты на v2 (при появлении спроса)

- **Интеграции с трекерами (Jira/Linear)** — вынесены из v1: расширение скоупа
  без ясной обучающей ценности для Go-ядра. Кандидат на v2, если появится спрос.

## 4. Excluded — Never (сознательно вне скоупа навсегда)

Эти пункты противоречат базовым принципам проекта («$0 инфра автору», «минимум
общения», BYOK, приватность) и не рассматриваются даже в отдалённой перспективе:

- **Свой веб-UI / SaaS / сервер / хостинг ключей** — противоречит «$0 инфра»
  и «минимум общения». Продуктовость — в качестве и governance CLI/Action,
  не в облачном сервисе.
- **Мультипользовательские аккаунты, биллинг, подписки внутри самого тула** —
  если понадобится монетизация Action, это берёт на себя GitHub Marketplace.
- **Проксирование LLM через свой бэкенд** — чужие токены + расходы + приватность,
  конфликтует с BYOK.
- **GUI-приложение** — это отдельные проекты портфеля (Tauri/IntelliJ).
- **Автопостинг сводок куда-либо, кроме PR-комментария через собственный Action**
  (Slack/email-интеграции) — оставляем пользователю через pipe/JSON-вывод в его
  собственную автоматизацию.
- **Телеметрия / «звонки домой»** — принципиально нет (доверие + приватность диффов).
