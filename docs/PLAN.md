## 1. Что это

`gitl` (git-log-lens) — CLI/CI-инструмент на Go, который читает git-историю
репозитория (коммиты, диффы, статистику за период или диапазон) и через LLM-API
превращает её в структурированный инженерный артефакт:

- **AI-ревью** диапазона/PR с **машиночитаемым риск-скорингом** (`low|medium|high`)
  для гейтинга в CI;
- **changelog** в стиле Keep a Changelog с группировкой по conventional-commits;
- **мульти-репо digest** — агрегированная сводка активности по нескольким сервисам.

Это не «сервис» и не «SaaS»: чистый CLI-бинарник плюс GitHub Action-обёртка.
Ни сервера, ни БД, ни хостинга ключей.

## 2. Зачем (и почему именно так)

Проект — **соло pet-проект с приоритетом обучения Go**, но с сознательно поднятой
продуктовой планкой: цель не «минимальный скрипт для себя», а **инструмент,
который не стыдно поставить в CI реального проекта и предложить команде**.

Учебная нагрузка на стандартную библиотеку сохраняется полностью:
структура проекта, `os/exec`, ручной `net/http`-клиент с retry/стримингом,
горутины/каналы, table-driven и golden-тесты, кросс-компиляция. Вокруг этого
ядра выстроен более серьёзный продуктовый контур (CI-гейтинг, governance,
подписанные релизы).

Ключевые принципы:

- **Нулевая инфраструктура для автора** — расходы ≈ $0/мес, никакого сервера.
- **BYOK + мультипровайдерность как продуктовое требование** (OpenAI-совместимый
  API, Ollama, Azure OpenAI), а не «приятный бонус».
- **Governance для командного использования** — repo-level shareable-конфиг,
  cost-guard, версионированная JSON-схема вывода.
- **Минимум общения от автора** — самообслуживание (Marketplace, документация,
  SemVer), а не саппорт-чат.
- **Без телеметрии и «звонков домой»** — принципиально (доверие + приватность диффов).

## 3. Для кого

- **Прежде всего — сам автор**: dogfooding на собственных репозиториях (лучший
  индикатор полезности).
- Разработчики и команды, которым нужен AI-ревью PR / генератор changelog /
  дайджест активности в CI без построения собственного сервиса.
- Команды с требованиями к резидентности данных (через Ollama / Azure OpenAI).

## 4. Ключевые архитектурные решения (верхнеуровнево)

| Решение | Кратко | Подробности |
|---|---|---|
| Язык | Go 1.22+, упор на stdlib | §4.1 TECHNICAL_PLAN |
| CLI-фреймворк | `spf13/cobra` + `spf13/viper` | §4.1 |
| Git-доступ | системный `git` через `os/exec` (не `go-git`) | §4.3 — обучающий эффект + свой парсер |
| LLM-доступ | ручной `net/http`-клиент, **без SDK** | §4.1 — тренировка retry/стриминга |
| Мультипровайдерность | явное поле `provider` (openai/ollama/azure_openai) | контракт, не угадывание по `base_url` |
| Конфиг | два уровня: repo `.gitl.yaml` > личный `~/.config/gitl/config.yaml` | §4.4 |
| Offline-режим | детерминированная эвристика без сети | dev/CI без ключа + fallback |
| Структура | `cmd/` + `internal/`, **без** `pkg/` | §4.2 — «не строй библиотеку, пока не просят» |
| Релиз | `goreleaser` + `cosign`/Sigstore keyless + `SHA256SUMS` | supply-chain-доверие в MVP |

## 5. Этапы (от 0 до конца MVP)

Терминология репозитория — **«Этап N»** (не «недели»). Порядок важнее сроков.
Детализация действий/deliverable/критериев — в
[`TECHNICAL_PLAN.md` §5](./TECHNICAL_PLAN.md).

| Этап | Название | Однострочное описание |
|---|---|---|
| **0** | Bootstrap | Репозиторий, окружение, документация, CI-скелет, зависимости, каркас кода. |
| **1** | Validation spike + каркас CLI | Снять риск №1 (годится ли AI-ревью диффа) на `review <range>`; cobra/viper-каркас, двухуровневый конфиг, offline-режим, первые тесты парсера. |
| **2** | LLM-клиент + мультипровайдерность + риск-скоринг + cost-guard | Надёжный `net/http`-клиент (retry+backoff), три провайдера как контракт, риск-скоринг, версионированный JSON, `--dry-run`/`--max-cost-usd`. |
| **3** | Полный набор команд + мульти-репо digest | `changelog`, `digest --days=N`, мульти-репо `digest --repos=...` (worker-pool), golden-тесты рендера. |
| **4** | GitHub Action | `action.yml`, постинг ревью комментарием к PR, check status по риск-скорингу, secrets-based ключ. |
| **5** | Релизный инжиниринг | `goreleaser` (3 ОС × 2 арх), `cosign` keyless-подпись, `SHA256SUMS`, `version` с build-provenance. |
| **6** | Marketplace + хардненинг + документация | Публикация Action, edge-case фикстуры, полная документация. **Конец MVP.** |

## 6. После MVP (сознательно отложено)

Подробно — в [`POST_MVP_PLAN.md`](./POST_MVP_PLAN.md). Кратко отложено:

- **Стриминг** ответа LLM в терминал (токен-за-токеном).
- **TUI** (`digest --tui` на bubbletea) поверх готовых артефактов.
- **Кэш** ответов LLM по хэшу диапазона (`~/.cache/gitl/`).
- Кастомные шаблоны промптов/вывода (`text/template`).
- Расширенный горутинный worker-pool сбора диффов.
- **Полный SLSA provenance** (следующий шаг после базового cosign-подписания).
- Интеграции с трекерами (**Jira/Linear**) — кандидат на v2 при спросе.
- **Платный тир GitHub Marketplace Action** — опциональная монетизация (биллинг
  на стороне GitHub).

Явно **Never** (вне скоупа навсегда): свой веб-UI/SaaS/сервер/хостинг ключей,
проксирование LLM через свой бэкенд, мультипользовательские аккаунты/биллинг
внутри тула, GUI-приложение, автопостинг в Slack/email из самого тула.

## 7. Definition of Done (MVP, кратко)

- Три команды (`review`, `changelog`, `digest`) работают на реальных репо;
  `digest` — в т.ч. мульти-репо.
- `review` даёт машиночитаемый риск-скоринг; `--fail-on=high` гейтит по exit code.
- JSON-вывод версионирован (`schema_version`) и задокументирован.
- Repo-level `.gitl.yaml` переопределяет личный конфиг с задокументированным приоритетом.
- `--dry-run`/`--max-cost-usd` протестированы и реально предотвращают перерасход.
- Три провайдера (OpenAI-совместимый, Ollama, Azure OpenAI) явно протестированы.
- GitHub Action опубликован в Marketplace и прогнан на реальном PR.
- Релиз подписан cosign, `SHA256SUMS` приложены, верификация задокументирована.
- Offline-режим: полноценная разработка и CI без ключа и сети.

## 8. Бюджет

Разовые расходы ≈ $0, ежемесячные ≈ **$0/мес** (нет сервера/БД/хостинга).
Токены LLM платит пользователь своим ключом (BYOK). Для dev-цикла:
offline-режим + локальная Ollama покрывают почти всё; тестовый прогон против
Azure OpenAI — единицы $ разово. С колоссальным запасом укладывается
в лимиты $500 разово / $50–70 в мес.
